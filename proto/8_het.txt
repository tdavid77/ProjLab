8. Hét: Prototípus Részletes Tervei

Ez a dokumentum a prototípus részletes terveit tartalmazza. A tervezés célja, hogy a programozó minden szükséges információval rendelkezzen az implementációhoz anélkül, hogy kitalálnia kellene bármit. A dokumentum három fő részre oszlik: objektumok és metódusok tervei, tesztek részletes tervei, valamint tesztelést támogató programok tervei.

1. Objektumok és Metódusok Tervei

Az alábbiakban minden releváns osztályt és interfészt részletesen leírunk. Minden metódus (private, protected, public) szerepel, algoritmusokkal, pszeudokóddal vagy activity diagrammal. Ha az objektumnak van állapota, akkor state-chart is mellékelve.

1.1. Jarmu osztály

Állapot: nincs (absztrakt osztály).

Metódusok:
- public void halad(Sav sav): Mozgatja a járművet a sávon. Algoritmus: Ellenőrzi, hogy a sáv szabad-e. Ha igen, beállítja a jármű pozícióját a sávra. Pszeudokód: if (sav.isSzabad()) { this.pozicio = sav; sav.setJarmu(this); }
- protected boolean isMozogKepes(): Ellenőrzi, hogy a jármű képes-e mozogni (alapértelmezetten true). Algoritmus: Visszaadja true-t.
- private void logMozgas(): Naplózza a mozgást. Algoritmus: Kiírja a konzolra: "Jármű mozog".

1.2. Auto osztály (örököl Jarmu-tól)

Állapot: nincs.

Metódusok:
- public void halad(Sav sav): Override. Algoritmus: Szülő metódus hívása, majd ha a sáv jeges, csúszás kockázata. Pszeudokód: super.halad(sav); if (sav.isJeges()) { if (random() < 0.1) baleset(); }
- private void baleset(): Baleset kezelése. Algoritmus: Növeli a balesetek számát globálisan.

1.3. Busz osztály (örököl Jarmu-tól)

Állapot: nincs.

Metódusok:
- public void halad(Sav sav): Override. Algoritmus: Szülő hívása, majd ha végállomás, fizetés. Pszeudokód: super.halad(sav); if (sav.isVegallomas()) { fizet(); }
- private void fizet(): Bevétel növelése. Algoritmus: Globális bevétel += 100.

1.4. Hokotro osztály (örököl Jarmu-tól)

Állapot: fej (Fej típusú).

State-chart: Állapotok: FejNincs, FejVan. Átmenetek: fej felszerelése -> FejVan, fej eltávolítása -> FejNincs.

Metódusok:
- public void fejCsere(Fej ujFej): Fej cseréje. Algoritmus: Ha telephelyen van, eltávolítja a régi fejet, felszereli az újat. Pszeudokód: if (isTelephelyen()) { this.fej = ujFej; log("Fej cserélve"); }
- public void takarit(Sav sav): Takarítás a fejjel. Algoritmus: Ha van fej, hívja a fej takarit metódusát. Pszeudokód: if (fej != null) fej.takarit(sav);
- protected boolean isTelephelyen(): Ellenőrzi, hogy telephelyen van-e. Algoritmus: Visszaadja a pozíció alapján.

1.5. Fej interfész

Metódusok:
- public void takarit(Sav sav): Absztrakt takarítás.
- public boolean isKesz(): Ellenőrzi, hogy kész-e a fej.

1.6. SoszoroFej osztály (implementálja Fej-et)

Állapot: soKeszlet (int).

State-chart: Állapotok: Kesz, Ures. Átmenetek: töltés -> Kesz, fogyasztás -> Ures.

Metódusok:
- public void takarit(Sav sav): Só szórása. Algoritmus: Ha van só, szór sót, olvasztja a havat. Pszeudokód: if (soKeszlet > 0) { sav.setSo(true); soKeszlet--; }
- public void toltsSo(int mennyiseg): Só töltése. Algoritmus: soKeszlet += mennyiseg.
- private boolean isUres(): Ellenőrzi, hogy üres-e. Algoritmus: return soKeszlet == 0;

1.7. SarkanyFej osztály (implementálja Fej-et)

Állapot: kerozinKeszlet (int).

State-chart: Hasonló a SoszoroFej-hez.

Metódusok:
- public void takarit(Sav sav): Tűzokádás. Algoritmus: Ha van kerozin, takarít, fogyaszt. Pszeudokód: if (kerozinKeszlet > 0) { sav.takarit(); kerozinKeszlet--; }
- public void toltsKerozin(int mennyiseg): Töltés. Algoritmus: kerozinKeszlet += mennyiseg.

1.8. Sav osztály

Állapot: horeteg (int), jeg (boolean), so (boolean).

State-chart: Állapotok: Szaraz, Havas, Jeges. Átmenetek: havazás -> Havas, autó áthaladás -> Jeges, só -> Szaraz.

Metódusok:
- public void havazas(): Havazás. Algoritmus: horeteg += 1; if (horeteg > 2) jeg = true;
- public void autotAthalad(): Autó áthaladás. Algoritmus: if (horeteg > 0) jeg = true;
- public void sotSzor(): Só szórása. Algoritmus: so = true; jeg = false;

1.9. Uthalozat osztály

Állapot: savok (lista).

Metódusok:
- public void havazas(): Minden sávon havazás. Algoritmus: for each sav in savok: sav.havazas();
- public boolean isJatekVege(): Játék vége ellenőrzés. Algoritmus: Ellenőrzi buszokat és baleseteket. Pszeudokód: if (buszokMozogKepes() && balesetek < 5) return false; else return true;

1.10. JatekosManager osztály

Metódusok:
- public void kezeldBuszokat(): Buszok kezelése. Algoritmus: Minden buszt mozgat.
- public void kezeldTakaritokat(): Takarítók kezelése. Algoritmus: Minden hókotrónak takarít.

2. Tesztek Részletes Tervei

A tesztek részletes leírása a teszt nyelvén. Pályák egyszerű szöveges leírással, bemenet és elvárt kimenet megadása.

2.1. Teszt: Busz végállomásra ér

Pálya: Egy sáv, végállomással.

Bemenet: 4\nI\n (menü 4, végállomás igen)

Elvárt kimenet: "Busz haladása és fizetés" -> "Végállomás? (I/N): I" -> "Bevétel növelve: 100" -> log sorok.

2.2. Teszt: Sószóró fej működése

Pálya: Havas sáv.

Bemenet: 7\n (menü 7)

Elvárt kimenet: "Sószóró fej működése" -> "Só kihelyezve" -> sáv állapot változás log.

2.3. Teszt: Játék vége

Pálya: Több sáv, buszokkal.

Bemenet: 10\nN\nN\n (menü 10, válaszok nem)

Elvárt kimenet: "Játék vége check" -> "Játék folytatódik".

3. Tesztelést Támogató Programok Tervei

3.1. Egy teszt lefuttatása: java skeletonprogram.SzkeletonProgram --test=N --input=input.txt --output=output.txt

3.2. Tesztek lefuttatása: java skeletonprogram.SzkeletonProgram --all > all_output.txt

3.3. Összehasonlítás: diff expected_output.txt actual_output.txt (Linux/Windows diff parancs). Ha külön program kell, egy egyszerű shell script: compare.sh, ami diff-et futtat és exit kóddal jelzi az eredményt.
